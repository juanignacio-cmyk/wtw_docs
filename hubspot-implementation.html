SECTION 3

Stage Gates — Required Properties

What MUST be filled before a deal can enter each stage

Configure these as required properties per stage in HubSpot under Pipeline Settings. Properties marked with a filled checkbox are hard gates — workflows will block stage entry without them.

Stage 0 — MQL

INBOUND ONLY
  • Contact Email
  • Company Name
  • Lead Source = "Inbound"
  • Lead Source Detail (which form/page)

SLA: 24hr first response, 48hr to move or disqualify

Stage 1 — Prospecting

OUTBOUND
  • Deal Name (format: "Company - DV/MS - Year")
  • B2B Source (dropdown)
  • ICP Tier (Tier 1 / 2 / 3)
  • Deal Owner
  • Event Source (if B2B Source = Trade Show)

SLA: First touch within 5 business days

Stage 2 — Outreach

ENGAGEMENT
  • All from Prospecting
  • Contact Role (dropdown) on primary contact
  • Next Step (text)

SLA: 3-6 touches over 14 days. Max 21 days — auto-move to Circle Back if no response.

Stage 3 — SQL (Sales Qualified Lead)

QUALIFICATION GATE
  • All from Outreach
  • SQL Checklist — Right Person = YES
  • SQL Checklist — Budget Exists = YES
  • SQL Checklist — Timing = YES
  • SQL Checklist — Interest = YES
  • Champion Type (Strong / Weak / Not a Champion)
  • Buyer Persona (dropdown)
  • Country
  • Deal Type B2B (DV / MS / DV+MS / Airports / Hotels / Other)
  • Estimated Deal Amount

HARD GATE: Cannot enter SQL without ALL 4 checklist items = Yes. SLA: Demo booked within 14 days.

Stage 4 — Demo

PRESENTATION
  • All from SQL
  • Demo Date
  • Number of Attendees
  • Decision Maker Present? (yes/no)

SLA: Quote sent within 3 business days post-demo

Stage 5 — Negotiation

PROPOSAL
  • All from Demo
  • Amount (required, no $0)
  • Close Date (required)
  • Number of Listings/Maps
  • Forecast Category (Commit / Best Case / Pipeline)
  • Deal Probability (%)

HARD GATE: Cannot enter without Amount + Close Date. SLA: Max 30 days, weekly check-in.

Stage 6 — Closing Soon

CONTRACT
  • All from Negotiation
  • Contract Sent Date
  • Expected Payment Date

SLA: Max 14 days. Slack alerts at day 7 and 10. Daily alerts after day 14.

Stage 7 — Deal Closed Won

CELEBRATION
  • All from Closing Soon
  • Closed Won Reason (dropdown)
  • Annual Contract Value
  • Number of Listings/Maps (final)

Auto-triggers: Contact lifecycle → Customer, CS handoff workflow, renewal deal creation.

Stage 8 — Closed Lost

REQUIRED REASON
  • Closed Lost Reason (REQUIRED dropdown)
  • Closed Lost Detail (optional text)
  • Reopen Eligible? (yes/no)

HARD GATE: Cannot close lost without selecting a reason. If Reopen Eligible = yes, auto-creates follow-up task in 90 days.

Stage 9 — Circle Back Later

NURTURE
  • Circle Back Reason (dropdown)
  • Circle Back Date (when to re-engage)
  • Last Stage Before Circle Back (auto-set)

RULES: Auto-task on Circle Back Date. If date passes + 30 days no activity → auto-Lost. Max 2 circle-backs per deal.

SECTION 4

Workflows to Build

17 automation rules — exact trigger/action logic for HubSpot Workflows

Build these in Automation → Workflows → Deal-based. Each rule includes the exact IF/THEN logic. Workflows are grouped by function.

SG

Stage Guards — Block Invalid Transitions

1 SQL Entry Gate

TRIGGER: Deal stage moves → SQL IF: sql_check_right_person != Yes OR sql_check_budget_exists != Yes OR sql_check_timing != Yes OR sql_check_interest != Yes THEN: Revert deal to previous stage Send internal notification to deal owner: "Deal [deal name] missing SQL qualification. Complete all 4 checklist items."
Critical Deal-based workflow

2 Negotiation Entry Gate

TRIGGER: Deal stage moves → Negotiation IF: amount = 0 OR amount is empty OR closedate is empty THEN: Revert deal to previous stage Send notification: "Cannot enter Negotiation without Amount and Close Date."
Critical Deal-based workflow

3 Closed Lost Reason Gate

TRIGGER: Deal stage moves → Closed Lost IF: closed_lost_reason is empty THEN: Revert deal to previous stage Send notification: "Must select a Closed Lost Reason before closing."
Critical Deal-based workflow

4 Circle Back Date Gate

TRIGGER: Deal stage moves → Circle Back Later IF: circle_back_date is empty THEN: Revert deal to previous stage Send notification: "Set a Circle Back Date before moving to Circle Back." ALSO: Increment circle_back_count by 1 Set last_stage_before_cb to previous stage value IF: circle_back_count > 2 THEN: Send alert: "Max 2 circle-backs reached. Force to Won or Lost."
Critical Deal-based workflow
TA

Time-Based Alerts — SLA Enforcement

5 MQL — 24hr Stale Alert

TRIGGER: Deal is in stage MQL AND: Last activity date > 24 hours ago THEN: Send email notification to deal owner Subject: "MQL Stale: [deal name] — no activity in 24hrs"
SLA Delay: 24hrs after stage entry

6 Prospecting — 7-Day Stale Alert

TRIGGER: Deal is in stage Prospecting AND: Last activity date > 7 days ago THEN: Send notification to deal owner "Prospecting deal [deal name] has no activity in 7 days."
SLA Delay: 7 days after stage entry

7 Outreach — 14-Day Warning + 21-Day Auto-Move

TRIGGER: Deal is in stage Outreach BRANCH 1 — Day 14: IF: No meeting booked after 14 days THEN: Alert owner: "No meeting booked. 7 days remaining." BRANCH 2 — Day 21: IF: Still no response after 21 days + 6 logged activities THEN: Auto-move deal to Circle Back Later Set circle_back_reason = "No Response" Set circle_back_date = today + 90 days
SLA Auto-move

8 SQL — 7-Day Alert + 30-Day Auto-Circle Back

TRIGGER: Deal is in stage SQL BRANCH 1 — Day 7: IF: No demo booked (demo_date is empty) THEN: Alert: "SQL deal [name] — 7 days, no demo scheduled." BRANCH 2 — Day 30: IF: No progress (no stage change, no demo_date) THEN: Auto-move to Circle Back Later
SLA Auto-move

9 Negotiation — Weekly Monday Alert + 45-Day Auto-Move

TRIGGER: Deal is in stage Negotiation RECURRING — Every Monday: IF: Last activity > 7 days ago THEN: Alert owner: "Negotiation deal [name] — no activity this week." BRANCH — Day 45: IF: 45 days in Negotiation with no stage change THEN: Auto-move to Circle Back Later BRANCH — Close Date Passed: IF: closedate < today THEN: Alert: "Close date passed — update required."
SLA Auto-move Close date watch

10 Closing Soon — Escalating Alerts

TRIGGER: Deal is in stage Closing Soon Day 7: Alert owner via Slack + email Day 10: Alert owner + manager via Slack + email Day 14+: Daily alert to owner + manager "Contract unsigned for [X] days. Escalate."
SLA Escalation

11 Circle Back — Re-engage + Auto-Lost

TRIGGER: Deal is in stage Circle Back Later BRANCH 1 — Circle Back Date arrives: THEN: Create task for deal owner: "Re-engage [deal name]" Send notification reminder BRANCH 2 — Circle Back Date + 30 days: IF: No activity since Circle Back Date THEN: Auto-move to Closed Lost Set closed_lost_reason = "No Response - Ghosted" QUARTERLY: Report of all Circle Back deals for review
SLA Auto-move Quarterly review
LC

Lifecycle Syncs — Contact Status Updates

12 Deal → SQL: Contact Lifecycle = SQL

TRIGGER: Deal stage changes to SQL THEN: Update associated contact lifecycle_stage = "Sales Qualified Lead"
Lifecycle

13 Deal → Won: Contact Lifecycle = Customer

TRIGGER: Deal stage changes to Deal Closed Won THEN: Update associated contact lifecycle_stage = "Customer" Trigger CS handoff workflow Create renewal deal in "AV Existing Clients" pipeline
Lifecycle Triggers sub-workflow

14 Deal → Lost: Do NOT Downgrade Contact

TRIGGER: Deal stage changes to Closed Lost THEN: Do NOT change contact lifecycle stage (never downgrade) IF: reopen_eligible = Yes THEN: Create follow-up task in 90 days for deal owner
Lifecycle Protect existing stage
DQ

Data Quality — Reporting & Hygiene

15 Weekly: Missing Required Fields Report

SCHEDULE: Every Monday at 9:00 AM ACTION: Generate report of all open deals where: - Stage = SQL but any sql_check field is empty - Stage = Negotiation but amount = 0 or closedate empty - Stage = any but Country is empty - Stage = any but ICP Tier is empty SEND TO: Sales manager + deal owners
Data Quality Scheduled report

16 Monthly: Circle Back Review

SCHEDULE: First Monday of each month ACTION: Generate list of all Circle Back deals: - Sort by circle_back_date (upcoming first) - Flag deals with circle_back_count = 2 (max reached) - Include last engagement score SEND TO: Sales team for review meeting
Data Quality Monthly cadence

17 Quarterly: Pipeline Hygiene Audit

SCHEDULE: First week of each quarter ACTION: Full pipeline audit report: - Stale deals (no activity > 30 days, by stage) - Missing data audit (% fill rate per required property) - Conversion rates per stage (vs. targets) - Average days per stage (vs. SLA) - Forecast accuracy: predicted vs. actual close SEND TO: Sales leadership + VP Sales
Data Quality Quarterly cadence

Migration Plan

3-week plan to triage 923 stuck SQL deals and go live with Pipeline v2

Week 1 Triage Score & bucket 923 deals
Week 2 Enrich Fill required properties
Week 3 Go Live Activate, move, train
Phase 1 — Triage (Week 1) 923 SQL deals
  1. Export all 923 SQL deals from HubSpot to spreadsheet
  2. Score each deal on triage criteria:
    • Has company name? (+1)
    • Has primary contact? (+1)
    • Has deal amount? (+1)
    • Any activity in past 90 days? (+1)
  3. Bucket into 4 groups:
~50-100
Keep in SQL
Recent activity + looks qualified
~200-300
Move to Circle Back
Had engagement but stalled
~400-500
Move to Closed Lost
No activity >6 months, no data
~100-200
Move to Prospecting
Never really qualified
Phase 2 — Data Enrichment (Week 2) Property fill
  1. "Keep in SQL" deals — fill ALL required SQL properties:
    • Country, ICP Tier, Champion Type, Buyer Persona
    • All 4 SQL checklist items
    • Deal Type B2B, estimated amount
  2. "Circle Back" deals — set Circle Back Date (batch to next quarter start)
  3. "Closed Lost" deals — set reason as "Historical - Unqualified" (temporary option to add, then archive later)
  4. "Prospecting" deals — assign to SDR for enrichment queue
Phase 3 — Go Live (Week 3) Launch day
  1. Activate new pipeline stages in HubSpot
  2. Execute bulk moves — move all triaged deals to their new stages
  3. Turn on automation rules — all 17 workflows (stage guards first, then alerts)
  4. Train team on new process — 1hr session covering:
    • New stage definitions and SLAs
    • Required properties per stage
    • What happens when you skip fields (gates block you)
    • Circle Back rules (max 2)
  5. 30-day parallel run — keep old stages visible but locked, remove after 30 days

Risk Mitigation

Data Loss During Migration

Mitigation: Full export backup before any bulk moves. Use HubSpot "Archive" not "Delete" for properties. 30-day parallel run before removing old stages.

Team Resistance to New Process

Mitigation: Involve top performers (Arturo, Philippe) in design. Show them what they win: less clutter, better forecasting, automatic alerts that help them.

Workflow Errors Blocking Deals

Mitigation: Activate stage guards first (week 3, day 1). Monitor for 48hrs before enabling auto-move workflows. Have an "emergency off" plan for each workflow.

Valid Deals Incorrectly Closed

Mitigation: The "Keep in SQL" bucket gets manual review by sales team, not auto-moved. Reopen Eligible flag on lost deals allows recovery.

Historical Data Skews Reports

Mitigation: Use "Historical - Unqualified" as a distinct lost reason. Filter it from pipeline conversion reports. Set a "migration date" property to separate pre/post data.

Automation Creates Noise

Mitigation: Start with email-only alerts (not Slack). Review alert volume after 1 week. Tune thresholds before adding Slack notifications.
